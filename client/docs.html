Veestream API Documentation

Welcome to the Veestream API documentation. Veestream is a video API that allows you to upload, stream, and manage your video content. This API is intended for anyone who wants to integrate video on demand feature into their web or mobile applications.

Base URL

The base URL for the Veestream API is http://veestream.tech.

Getting an API key

An API key is required to use the veestream API. To get an API key,register by providing your name, email and paying 1$ via paypal, credit card or debit card. After you signup, you will get an API key. It is recommended that you keep this API key safe and private. This API key must be provided in the request header when making calls to other API endpoints. After the payment, you will get an email from paypal with your transaction receipt. It is recommended that you keep this receipt safe for reference.

Endpoints

Upload Video

This endpoint allows a user with an API key to upload one video at a time. When you signed up, an azure blob storage container is created for you. All the video files you upload will be stored on that container.

POST /video/upload
Headers

Use the following header in your request for the upload endpoint:

apikey: yourapikey
Body

Include name in the request body. This should be a string, the name for your file.

name:my Video
File Upload

To upload a file, send a POST request to the API endpoint and include the file as a parameter named "video".

The following example demonstrates how to upload a file using curl

curl -X POST -H "apiKey: b7642f0d6db02f383498e7f7edd353597ad8e5fe" -F "video=@/home/amschel/Downloads/honey.mp4" -F "name=My Video" http://veestream.tech/video/upload

The following example demonstrates how to upload a file using javascript

const apiKey = 'b7642f0d6db02f383498e7f7edd353597ad8e5fe';
	const fileInput = document.querySelector('#file-input');
	const formData = new FormData();
	formData.append('video', fileInput.files[0]);
	formData.append('name', 'My Video');
	
	fetch('http://veestream.tech/video/upload', {
	  method: 'POST',
	  headers: {
		'apiKey': apiKey,
	  },
	  body: formData
	})
	.then(response => response.json())
	.then(data => console.log(data))
	.catch(error => console.error(error));

The following example demonstrates how to upload a file in flutter

import 'dart:io';
		import 'package:http/http.dart' as http;
		
		final String apiKey = 'b7642f0d6db02f383498e7f7edd353597ad8e5fe';
		final String url = 'http://veestream.tech/video/upload';
		final String filePath = '/home/amschel/Downloads/honey.mp4';
		
		Future uploadFile() async {
		  final file = File(filePath);
		
		  final request = http.MultipartRequest('POST', Uri.parse(url));
		  request.headers.addAll({'apiKey': apiKey});
		  request.files.add(http.MultipartFile.fromBytes('video', await file.readAsBytes()));
		  request.fields['name'] = 'My Video'; // add a field to the request
		
		  final response = await request.send();
		  final responseJson = await http.Response.fromStream(response).then((res) => res.body);
		
		  print(responseJson);
		}

Stream Video

GET /video/:id

This endpoint creats a readable stream from your video file in azure to the client and the video is streamed in chunks.When you upload a video, you will get a response with an _id property. use this _id as the request parameter. e.g 

GET /video/61f19d85d28a103995b1816c

Below is javascript code that can be used to make the rquest to te endpoint

const videoUrl = "http://veestream.tech/video/61f19d85d28a103995b1816c";
		const apiKey = "b7642f0d6db02f383498e7f7edd353597ad8e5fe";
		const videoTag = document.getElementById("my-video");
		
		const headers = {
		  "apiKey": apiKey
		};
		
		fetch(videoUrl, {
		  method: "GET",
		  headers: headers
		})
		.then(response => response.blob())
		.then(blob => {
		  const videoBlobUrl = URL.createObjectURL(blob);
		  videoTag.src = videoBlobUrl;
		})
		.catch(error => console.error(error));

Get Video Metadata

GET /video/:id/data

This allows you to get metadata about a particular video. We pass the id parameter to specify the video we are requesting data for. id is from _id field that each video has.

Below is an example curl command

curl -X GET -H "apikey: b7642f0d6db02f383498e7f7edd353597ad8e5fe" http://veestram.tech/video/61f19d85d28a103995b1816c/data

Below is the example code in javascript for this endpoint

		const axios = require('axios');

const url = 'http://veestram.tech/video/61f19d85d28a103995b1816c/data';

const headers = {
  apiKey: 'b7642f0d6db02f383498e7f7edd353597ad8e5fe',
};

axios.get(url, { headers })
  .then((response) => {
    console.log(response.data);
  })
  .catch((error) => {
    console.log(error.response.data);
  });

Generate Video Poster

A video poster is generated by taking a screenshot of the video from 10-12secs. It is of type image/png. Below is the example curl command for this. 

curl -X GET \
		http://veestream.tech/video/61f19d85d28a103995b1816c/poster \
		-H 'apikey: b7642f0d6db02f383498e7f7edd353597ad8e5fe'

When you make a request to /video/:id/data, you get a poster as part of the response. This poster in the response is basically a url to this endpoint. When you add the poster as the src attribute of an image, what happens is that the poster endpoint is called to generate a poster.

Generate Video GIF

A video gif is generated using ffmpeg which takes a video and generates a 5 second gif. 


		curl -X GET \
		http://veestream.tech/video/61f19d85d28a103995b1816c/gif \
		-H 'apikey: b7642f0d6db02f383498e7f7edd353597ad8e5fe'

When you make a request to /video/:id/data, you get a gif too as part of the response. This gif in the response is basically a url to this endpoint. When you add the gif as the src attribute of an image, what happens is that the gif endpoint is called to generate a gif.

This endpoint generates a GIF for the specified video. Remember to include the apikey header in your request.
